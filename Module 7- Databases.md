**Amazon Relational Database Service (Amazon RDS)**

Amazon RDS is a _managed_ relational database service that handles routine database tasks such as backups, patching, and hardware provisioning. Amazon RDS supports multiple database instance class types that optimize for memory, performance, or input/output (I/O).Multi-AZ deployment and automated backups

**Amazon Aurora**

Aurora provides a high-performance, highly available storage architecture that offers up to five times the throughput of standard MySQL . Aurora is a managed relational database designed to help reduce unnecessary I/O operations . Multiple Availability zones

---

**NoSQL AWS Database - Amazon DynamoDB**

fast and predictable performance for both document and key-value data structures. Automatic Scaling

Use case : gaming platforms,financial service applications, and mobile applications

---

**Amazon ElastiCache -** Cache memory for RDS , use the same Redis, Valkey, or Memcached tools

---

**Amazon DocumentDB -** is fully managed service designed to handle semistructured data, which is information that doesn't conform to rigid relational schemas. Amazon DocumentDB is a MongoDB-compatible database, so it manages JSON-like documents with dynamic schemas.

**AWS Backup** addresses the challenge of managing separate backup solutions for different AWS services by providing a unified, centralized backup service across the AWS Cloud.

**Amazon Neptune** is a fully managed graph database service designed for highly connected datasets. It supports popular graph query languages like Apache TinkerPop Gremlin and W3C's SPARQL, making it ideal for applications such as social networks, recommendation engines, fraud detection, and knowledge graphs.
